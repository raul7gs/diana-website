Aircraft Design Charts

These charts are specially used for aircraft design. All these charts are only available if the inputs are CPACS files

Aircraft Design Kit and Combine

To generate an aircraft design chart, you can use the so-called aircraft design kit. This kit contains 12 different charts covering files such as geometry, aerodynamics or performance, among others. The currently supported charts are mass breakdown, lifting surface geometry (wing , HTP and VTP), the aerodynamic polar (cd vs cl and L/D vs cl), the payload range diagram, the 3-D of the aircraft, the power breakdown, the center of gravity chart, the specific block energy and finally the trajectory plots. Many of these charts have their additional settings, which will be explained in more details along the documentation.

To create the chart any chart is really simple, you need to click on the desired chart type and in the aircraft desired to be included in the chart. Multiple charts can me generated for multiple aircraft simultaneously. If all aircraft are wanted to be included in the same chart, the combine checkbox can be used.

If an aircraft file does not have enough information to generate a specific diagram, the corresponding chart checkbox will appear as disabled.

MTOW/OEM Breakdown

This chart allows you to observe the mass distribution of the different components within the aircraft in a sunburst chart. They are divided in three main sections which are OEM (operative empty weight), fuel and payload.

The OEM section is subdivided in all the different structural components of the aircraft, including wings, systems or fuselage among others. The fuel is subdivided in block and reserve and the payload in passengers and cargo

To obtain more details of the different components of the OEM, it is possible to get a more detailed view by generating the OEM breakdown of the aircraft. This can be done when clicking on the corresponding checkbox in a selector (which is the default option).

In the case of the MTOW chart (mass take off mass) breakdown the suer need to select a specific mission, although by default the design mission is selected

Finally, both charts can be represented as bar charts in the settings button, giving additional insights of the mass distribution. It is also the plot type automatically chosen for the comparison of multiple aircraft (when the combined option is active)

Lifting Surface Geometry Charts

These charts allow to observe the geometry of the different lifting surfaces of the aircraft. These are the wing, the horizontal tail plane (HTP) and the vertical tail plane

Two additional settings exist in these charts. These are the option to observe the surface movables (flaps, elevator, rudder,....) and the option in the comparison to set a common origin for the different surfaces to compare sizes in a clearer manner.

Payload-Range Diagram

CPACS nodes necessary: /cpacs/vehicles/aircraft/model/analyses/trajectories

This chart allows users to study how the mission payload mass and range are related. Specilly interesting is to observe the maximum range of the aircraft (bottom right). In the settings is also possible to add the different calculated missions as markers, allowing to determine if the satisfied the requirements or not.

Aerodynamic Polar

These charts are used to evaluate the aerodynamic performance of the aircraft. Two different charts are permitted. The first one, on the left, is the aerodynamic polar (lift coefficient versus drag coefficient). The second chart, on the right, shows the aerodynamic efficiency versus the lift coefficient.

In both cases in the settings is possible to add the desired range for cruise (with dash lines) and the actual cruise performance of the aircraft (with a croos mark), allowing to check if the aircraft satisfies the requirements or not.

3D geometry

This chart allows to observe the aircraft in 3D. Four different views are allowed: isometric, front, side and top. It is also possible to set the camera totally free, allowing the user to obtain any desired view of the aircraft. Additionaly, in the front, side and top views it is possible to add dimensions automatically to the aircraft, as shown on the right of picture below.

CG plots

These plots represent the center of gravity (CG) distribution of the aircraft. The user can select from three different chart types, each displayed as a scatter plot showing the CG positions along the fuselage’s longitudinal axis (x-axis) and the height relative to the fuselage centerline (y-axis).

• Components CG: Displays the center of gravity of individual aircraft components such as the fuselage, wings, horizontal and vertical tailplanes (HTP and VTP), engines, and onboard systems.

• Fuel and Payload CG: Shows the CG positions for the fuel and payload, subdivided into cargo and passengers, along with the aircraft’s OEM (Operating Empty Mass) CG.

• Overall CG: Represents the total aircraft CG for different loading configurations: MTOM (Maximum Take-Off Mass), OEM + fuel, OEM + payload, and OEM only — illustrating the aircraft’s complete operational CG range.

Each point in the plot is displayed with a distinct color to represent different components or data categories. When comparing multiple aircraft, the plot uses different colors for each aircraft, allowing clear visual differentiation. Points belonging to the same aircraft share the same color but can be individually identified through Plotly’s hover functionality

SBER

The Specific Block Energy (SBE) vs. Range plot illustrates the aircraft’s energy consumption per passenger and per nautical mile across different mission ranges.

The tool filters all available aircraft missions according to payload and range, then groups them based on payload level. The user can then plot all missions with the same payload on an SBE vs. Range chart to analyze how mission range affects energy consumption for a given payload configuration.

As with other plots, this feature also supports comparing different aircraft concepts, allowing direct assessment of energy efficiency across multiple designs or configurations.

Trajectories

The Trajectory Plot displays all available flight trajectories for a set of aircraft, including low-speed trajectories, block missions, and reserve missions. Each trajectory consists of a series of variables computed at different time steps throughout the mission. Some examples are ground distance, Mach number, lift-to-drag ratio (L/D), aerodynamic coefficients, and thrust.

The user can select which variables to plot using two dropdown selectors: one for the x-axis variable and another for the y-axis variable. The y-axis supports displaying two variables simultaneously, allowing direct comparison against the same x-axis variable. By default, the ground distance is selected as the x-axis variable. The chart type is a line chart.

The plot also supports multiple aircraft, enabling easy comparison of equivalent missions across different aircraft designs.

Engine deck

The engine decks are data structures that store the engine performance parameters available for different operating conditions, defined by Mach number and altitude. Each deck contains information such as available thrust settings, turbine entry temperature (TET), fuel flow, NOx emissions, and other relevant performance data.

Two types of plots are available:

• SFC Bucket Plot: Displays the thrust-specific fuel consumption (SFC) of the engine as a function of thrust for a given flight condition. The user can select multiple altitudes and Mach numbers from the engine deck, and the plot will show all corresponding combinations with their respective SFC curves (“buckets”). This plot also supports multi-aircraft comparison, allowing users to evaluate fuel consumption differences between designs under identical flight conditions.

• Thrust Lapse Plot: Illustrates how available thrust varies with Mach number for different altitudes. The user can select one or more altitudes from the aero-performance map and visualize all curves together. As with the SFC plot, this view supports multiple aircraft, enabling direct comparison of thrust performance across different designs.

Power breakdown

CPACS nodes necessary: Vehicles/aircraft/model/systemArchitectures, Vehicles/aircraft/model/systems, Vehicles/aircraft/model/analyses/powerBreakdowns, Vehicles/systemElements

This plot visualizes the CPACS node powerBreakdowns as a Sankey diagram, which allows to visualize the various power flows exchanged between system elements, as seen in the figure below. A fix color coding is applied to the power flows: light blue for mechanical power, yellow for electric power, red for heat flow, dark blue for mass flow.

The user is asked to select which power breakdown to visualize, from a drop-down menu containing all powerBreakdown uIDs found in the CPACS file, or to visualize all powerBreakdowns at once though the all_systems option.

In the resulting plot, all system elements are draggable for custom placement.

Marks, filters and plot types

To further explore the different functionalities of the personalized plots GUI, we are going to use a dataset with around 2000 different military aircraft from the DLR. In this case, it has been decided to plot three key variables, which are the MTOW, the fuel mass and the wing span. According to literature, the most suitable chart for this example case was a scatter matrix.

Once the chart has been generated, it is possible to add additional variables though markers in the form of colour, shape and/or size. This can be observed in the picture below, where the drag coefficient has been added as a colour mark.

Filters can be applied both before or after the chart has been generated. These are interactive, so that the effects can be observed in the charts in real time. Right now, simple filters based exclusively on variables values are admitted. This means ranges for continuous filters and specific values for discrete variables. Multiple filters can be applied simultaneously

Finally, as mentioned before, if the data selected can be plotted in a different chart type, all the different options will appear coloured (as observed below). To change to a different plot type, it is only necessary to click ojn the corresponding picture. Correlation Matrix

This chart allows to visualize the relationship between the different variables. The scale goes from -1 to 1, where -1 means the variables are perfectly inverse proportional and 1 that they are perfectly positively correlated. In other words, negative values mean that when one variable increases the other tends to decrease, and positive values mean when a variable value increases the other variable does the same.

Values close to 0 mean there is no correlation between the variables. A colour scale is applied for a faster understanding. It is important to note that correlation matrices fail to accurately consider non linear relationships, being necessary for this more advanced methods such as SHAP (for more information consider the datat analysis section of the documentation).

Radar Chart

This is one of the most suitable charts for the comparison of a small quantity of designs for multiple variables simultaneously. In this example, different weights are compared for three similar aircarft designs.

For each variable, it is possible to see the difference for the different designs. In case that the different variables compared have different units or important differences in the scales, it it possible to normalize them in the settings button, simplifying the comparison process.

Box plot

Box plots are statistical charts that allow to understand the distribution of a certain variable along the design space. It is divided in two parts, the box and the whiskers. The box represents the quartiles 1 and 3, with the median (q2) in the middle. The whiskers represent the maximum and minimum variables to be considered normal. Every value outside of these limits is considered an outlier

If desired, it is possible to observe the design points by a checkbox found in the settings, as observed in the picture above

Dashboard

Once all the necessary charts have been generated, these can be combined together for decision-making in the dashboard tab, as shown below. The process to generate a dashboard is really simple. There is a selector in the top of the GUI with all the charts that have been generated, either aircraft design charts or personalized ones. The user just needs to click on the desired charts and they will be incorporated into the dashboard. To maintain an optimal balance between information density and readability needed for decision-making, users can generate new dashboards by clicking the blue addition button on the top left

Each dashboard is a white canvas, where the user can add as many charts as desired. Moreover, the user has full control over each chart size and positioning, through drag-and-drop functionality. The chart can be moved by clicking on the top right and its dimensions can be modified by clicking on the bottom right.

Dashboards also allow to add text and images. In the case of texts, a drag and drop element with modifiable text will be added. For images, an expandable asking for a picture to be uploaded will appear.

Finally, it is important to mention that the different charts within the app are interconnected. This means if one or more design points are filtered or selected in one chart, the same will happen in the other charts, as shown below.

Data analysis

When dealing with big datasets, it is necessary that the user has enough tools to discover trends within the data, as well as the most interesting points to perform a more detailed analysis. This is the purpose of the following methodologies encompassed by this section. All these functionalities are found in the data tab within DIANA on the right side of the data table

Pareto front

When there are multiple conflicting objectives to consider, Pareto fronts are commonly used. A Pareto front represents the set of optimal design solutions where improving one objective would require compromising at least one other. To show an example of the process within DIANA, we will consider a dataset with around 2300 different space launchers.

The first step is to include the objectives within DIANA. It is necessary to specify for each one rather if they are wanted to be minimized or maximized. In our example, we want to maximize the payload mass while minimizing the cost

The next step is to set the constraints, in case they exist. In our case, there were a structural and a volume constraint. It only accepts numerical constraints, both inequality and equality ones.

Finally, to see the Pareto front you just need to set the objectives in the axes of a 2D or 3D scatter plot, and add the Pareto front as a marker, as shown below.

Outliers detection

Outliers are data points that significantly differ from the rest of the dataset. They play a key role in data analysis, as they are often associated with errors, anomalies, or optimal designs. Therefore, analysts should dedicate time to investigating them. There are plenty of different methods to obtain these outliers, however, within DIANA there are three methods: Isolation Forest, DBSCAN and K means

The inputs required by each of them are found within DIANA GUI, which can be observed in the figure below. For more information about how these methods work, users can refer to the scikit-learn documentation

Once the outliers have been identified, a new boolean variable is generated, indicating whether each data point is an outlier or not (and, in some cases, to what degree). This variable can later be used as a marker. An example of this applied to a military aircraft dataset is shown below

Automatic Clustering

Automatic clustering is an unsupervised machine learning technique that automatically groups data into different clusters based on their similarities. This helps users uncover hidden patterns and insights within the dataset, such as previously unknown relationships between data points..

There are again different methods that can be used within DIANA to obtain these clusters, such as K means or Affinity propagation. For more infomration, users can refer to the scikit-learn documentation

Once the chosen algorithm has been executed, a new variable is obtained linking each design point to a certain cluster. This variable can be later used to observe the clusters, as shown in the figure below

Explainable IA

Data analysis allows us to uncover insights within a dataset, but in most cases, it does not explain the reasoning behind those insights. For example, it can detect outliers, but it cannot explain why certain data points are considered outliers. To bridge this gap, explainable AI (XAI) can be used, enabling designers not only to save time when trying to understand the machine’s results but also to build trust in them. These functionalities will soon be available in DIANA.
